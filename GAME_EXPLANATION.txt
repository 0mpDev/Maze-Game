────────────────────────────────────────────────────────────────────────────────
3. ALL PROJECT FILES AND THEIR PURPOSE
────────────────────────────────────────────────────────────────────────────────

    app.py
        The main Flask backend — the brain of the game.
        Handles all page routes and API endpoints.
        Coordinates maze generation, AI model, and session data.

    maze_generator.py
        Generates the maze using DFS (Depth First Search) algorithm.
        Also runs BFS to verify the maze is solvable and compute optimal path.
        Places rocks (obstacles) on path cells — passable but cost 1 life.

    model.py
        The Deep Learning model (Feedforward Neural Network / MLP).
        Architecture: 4 inputs → 16 → 8 → 3 outputs
        Takes player performance stats and predicts maze difficulty factors.
        At startup, automatically loads trained_weights.npz if present.
        Falls back to hand-crafted weights if trained weights not found.

    train_model.py
        One-time training script. Run with: py -3.13 train_model.py
        Generates 5000 synthetic gameplay sessions as training data.
        Trains the MLP using PyTorch (MSE loss + Adam optimizer + early stopping).
        Saves learned weights to trained_weights.npz.
        Also saves a training_plot.png showing loss curves.

    retrain.py
        Fine-tuning script. Run with: py -3.13 retrain.py
        Reads real player data from gameplay_data.json.
        Mixes real data (30%) with synthetic data (70%) to prevent forgetting.
        Fine-tunes the model from current trained_weights.npz at a lower LR.
        Run periodically after 5+ real players have completed the game.

    trained_weights.npz
        NumPy archive storing the 6 weight/bias arrays of the trained MLP.
        (W1, b1, W2, b2, W3, b3)
        Generated by train_model.py. Updated by retrain.py.
        Loaded by model.py at runtime — no PyTorch needed to run the game.

    gameplay_data.json
        Real player training data collected automatically during gameplay.
        Each entry stores level N stats (input to DL model) and level N+1
        performance (delayed feedback), used by retrain.py to improve the model.

    players.json
        Persistent storage file — the Hall of Fame.
        Every player who completes all 3 levels is saved here.
        Read by the /history page.

    requirements.txt
        Python packages needed to run the game:
            Flask      — web framework
            numpy      — used by the neural network model
            torch      — used only for training (train_model.py)
            matplotlib — used only for loss plot (optional)

4. COMPLETE GAME FLOW (STEP BY STEP)
────────────────────────────────────────────────────────────────────────────────

STEP 1 — LOGIN PAGE (/)
    Player opens the game in browser.
    Enters their name and birthday.
    Clicks "Set Sail!"
    Flask creates a SESSION storing:
        player_name, birthday, level=1, lives=3, stats=[]

STEP 2 — STORY PAGE (/story)
    Player reads the pirate story.
    Makes choices (Yes/No buttons).
    Speech synthesis reads the story aloud.
    Clicks "Let's Sail!" to start the game.

STEP 3 — GAME PAGE LEVEL 1 (/game)
    Browser loads game.html.
    JavaScript calls POST /api/get_maze.
    Flask calls maze_generator.py:
        - Generates a fixed 15x15 maze using DFS
        - Verifies it is solvable with BFS
        - Returns: grid array, start position, goal position
    JavaScript draws the maze on Canvas:
        - Gray rocky blocks = walls (cannot walk through)
        - Blue ocean = open paths (can walk through)
        - RED block labeled "P" = player (starts at top-left)
        - GREEN block labeled "G" = goal (bottom-right)
    Player presses arrow keys to move.
    Ocean ambient sound starts on first keypress.

STEP 4 — GAMEPLAY LOOP (inside game.js)
    Every arrow key press calls tryMove(direction):

        IF moving into a wall (gray rock block):
            → takeDamage() is called
            → Play crash sound (thud + life-lost melody)
            → lives = lives - 1
            → Show notification "Crashed into rock wall! -1 life"
            → Update heart display in HUD
            → 0.9 second invincibility (so one crash = only 1 life lost)
            → IF lives = 0: show Game Over overlay, play game over sound

        IF moving into open path (blue ocean):
            → Player moves to new cell
            → Play water splash sound
            → moves counter increases
            → Timer keeps running

        IF player reaches GREEN goal block:
            → Play win fanfare
            → Voice says "Yo Ho Captain! You found the treasure!"
            → Calls POST /api/level_complete with:
                  time_taken, mistakes, moves, optimal_moves, lives

STEP 5 — LEVEL COMPLETE API (/api/level_complete)
    Flask receives the stats.
    Calculates path efficiency = optimal_moves / actual_moves
    Saves stats to session.
    Resets lives to 3 for next level.

    Saves real training sample to gameplay_data.json:
        When level 2 finishes: level 1 stats (input) + level 2 stats (feedback)
        When level 3 finishes: level 2 stats (input) + level 3 stats (feedback)
        This delayed feedback is used by retrain.py to improve the model.

    Runs the NEURAL NETWORK (model.py):
        Input:  [time_norm, mistake_norm, inefficiency, lives_lost_norm]
        Hidden: 4 → 16 → 8 neurons (ReLU activation, specialised pathways)
        Output: 3 values (size_factor, rocks_factor, complexity_factor)
        Maps to: maze size, number of rocks, complexity for next level
    Stores next level parameters in session.
    IF level 3 just completed: saves player to players.json.
    Advances session["level"] to next level.

STEP 6 — LEVEL COMPLETE PAGE (/level_complete)
    Shows the player their performance stats.
    Shows AI adaptation message:
        Good player (high efficiency) → "Expect a HARDER challenge!"
        Average player               → "Expect a MODERATE challenge."
        Struggling player            → "Expect a GENTLER maze."
    Player clicks "Go on Another Hunt!" to play next level.

STEP 7 — REPEAT FOR LEVELS 2 AND 3
    Same gameplay, but maze is now AI-generated:
        Level 2: size decided by neural network (17-23 cells)
        Level 3: size decided by neural network (always ≥ Level 2 + 2)
    Smooth difficulty — no sudden jumps.
    Lives always reset to 3 at start of each level.

STEP 8 — WIN PAGE (/win)
    Shown after completing level 3.
    Fireworks animation plays.
    Triumphant fanfare plays.
    Voice congratulates the captain.
    Shows full stats for all 3 levels.
    Flask saves player to players.json (backup save, in case AJAX failed).
    Buttons: "Play Again" or "Pirate History"

STEP 9 — HISTORY PAGE (/history)
    Reads players.json.
    Shows all captains who completed the full game.
    Ranked with medals (Crown, Silver, Bronze for top 3).
    Shows each captain's name, date, total time, crashes, efficiency.

5. HOW THE MAZE GENERATION WORKS (DFS ALGORITHM)
────────────────────────────────────────────────────────────────────────────────

DFS = Depth First Search

The maze is a grid of cells. Each cell is either:
    0 = Wall (gray rock block — cannot walk through)
    1 = Path (blue ocean — can walk through)

Algorithm steps:
    1. Fill entire grid with walls (all 0s)
    2. Start at cell (1,1) — mark it as path (1)
    3. Push (1,1) onto a stack
    4. While stack is not empty:
           Look at current cell
           Pick a random direction (up/down/left/right)
           Move 2 steps in that direction (to skip one wall cell)
           If destination is unvisited:
               Carve the wall between (make it path = 1)
               Mark destination as path (1)
               Push destination onto stack
           Else: pop current cell off stack (backtrack)
    5. Result: a PERFECT maze — every cell is reachable from start

Solvability guarantee:
    After generation, BFS (Breadth First Search) runs from start to goal.
    If no path found (extremely rare): regenerates with a new seed.
    The optimal path length returned is used to calculate efficiency.


────────────────────────────────────────────────────────────────────────────────
6. HOW THE AI MODEL WORKS (NEURAL NETWORK / MLP)
────────────────────────────────────────────────────────────────────────────────

Model type: Feedforward Neural Network (Multi-Layer Perceptron)
Library: NumPy at runtime, PyTorch for training

Architecture:
    Input Layer  : 4 neurons
    Hidden Layer1: 16 neurons (ReLU activation)
    Hidden Layer2: 8 neurons  (ReLU activation)
    Output Layer : 3 neurons  (Sigmoid activation → clipped to [0.1, 0.9])

Inputs (after each level, all normalized to [0,1], higher = worse):
    [1] time_norm        = time_taken / 300
    [2] mistake_norm     = mistakes / 15
    [3] inefficiency     = 1 - (optimal_moves / actual_moves)
    [4] lives_lost_norm  = (3 - lives_left) / 3

Outputs (3 independent difficulty factors):
    [1] size_factor       → driven by time + efficiency → maps to maze grid size
    [2] rocks_factor      → driven by mistakes + lives  → maps to number of rocks
    [3] complexity_factor → driven by all signals       → maps to maze complexity

Hidden Layer 1 — 16 specialised neurons:
    Neurons  0-3 : Inverters     (fire HIGH when a metric is LOW = good performance)
    Neurons  4-7 : Overall perf  (uniform + per-metric weighted averages)
    Neurons 8-11 : Struggle detectors (fire HIGH when a metric is HIGH = bad)
    Neuron    12 : Fast + efficient combo detector
    Neuron    13 : Accurate + safe combo detector
    Neuron    14 : Slow + inaccurate combo detector
    Neuron    15 : Inefficient + lives-lost combo detector

Hidden Layer 2 — 8 neurons in 4 isolated pathways:
    H2[0,1] : TIME pathway       (receives time-specific H1 signals only)
    H2[2,3] : MISTAKES pathway   (receives mistake-specific H1 signals only)
    H2[4,5] : EFFICIENCY pathway (receives efficiency-specific H1 signals only)
    H2[6,7] : LIVES pathway      (receives lives-specific H1 signals only)

Output connections:
    size_factor       ← TIME pathway only
    rocks_factor      ← MISTAKES + LIVES pathways
    complexity_factor ← EFFICIENCY + MISTAKES + LIVES pathways

Training (train_model.py):
    - 5000 synthetic gameplay sessions generated as training data
    - Loss function : MSE (Mean Squared Error)
    - Optimizer     : Adam (lr=0.001, weight_decay=1e-4)
    - LR scheduler  : ReduceLROnPlateau (halves LR when val loss plateaus)
    - Early stopping: patience=20 epochs on validation loss
    - Train/Val/Test split: 70% / 15% / 15%
    - Weights saved to trained_weights.npz

Real data fine-tuning (retrain.py):
    - Collects real player data via gameplay_data.json
    - Label derivation (delayed feedback):
        size_factor_label       = 1 - time_norm_nextlevel
        rocks_factor_label      = 1 - mistake_norm_nextlevel
        complexity_factor_label = 1 - inefficiency_nextlevel
    - Mixes 30% real + 70% synthetic to avoid catastrophic forgetting
    - Fine-tunes at lower LR (5e-4) to preserve existing knowledge
    - Run after every 5+ new real player sessions

Difficulty mapping:
    Fast + efficient player → HIGH size_factor    → bigger maze next level
    Accurate + careful player → HIGH rocks_factor → more obstacles next level
    Overall good player → HIGH complexity_factor  → more complex layout

Level constraints (no sudden difficulty jumps):
    Level 2: size = 17 to 23   (always bigger than Level 1's fixed 15)
             rocks = 2 to 12
             complexity = 0.25 to 0.60
    Level 3: size >= Level2_size + 2    (always harder than Level 2)
             rocks >= Level2_rocks + 1
             complexity >= Level2_complexity + 0.10

7. SOUND EFFECTS (WEB AUDIO API — NO FILES NEEDED)
────────────────────────────────────────────────────────────────────────────────

All sounds are SYNTHESIZED in the browser using Web Audio API.
No MP3 or WAV files are needed.

    Move (splash)
        Bandpass-filtered noise burst
        Sounds like: water drip / soft splash

    Wall Crash (thud)
        Low sine wave + noise burst with fast decay
        Sounds like: dull wooden thud

    Life Lost (dramatic)
        Descending A-minor arpeggio: A → F → D → A
        Sounds like: dramatic "uh oh" melody

    Win / Level Complete (fanfare)
        Ascending C-major: C → E → G → C (high)
        Sounds like: triumphant trumpet fanfare

    Game Over (sorrowful)
        Descending A-minor melody: A → G → F → E → D
        Sounds like: sad defeat music

    Ocean Ambient (background loop)
        Two detuned sine oscillators (58Hz + 61.5Hz) = wave beating effect
        Low-pass filtered white noise = "shhhh" surf sound
        Slow LFO (0.18Hz) on noise volume = rhythmic wave swell
        Fades in over 3 seconds on first keypress

Voice Lines (Web Speech API):
    "Loading game... Yo Ho Captain!" — on game start
    "Yo Ho Captain! You found the treasure!" — on level complete
    "Oh no! The ocean has claimed you, Captain!" — on game over
    Story narration — on story page

Mute button (🔊/🔇) in HUD — fades all audio in/out smoothly.


────────────────────────────────────────────────────────────────────────────────
8. LIVES SYSTEM
────────────────────────────────────────────────────────────────────────────────

    Start of each level: 3 lives (always resets, not carried over)
    Crash into a wall (gray rock): -1 life
    Crash into maze border: -1 life
    0.9 second invincibility after each crash (prevents instant death)
    Lives = 0: Game Over — player must try the same level again
    Reaching goal: level complete regardless of lives remaining


────────────────────────────────────────────────────────────────────────────────
9. HOW PLAYER NAMES ARE SAVED TO HISTORY
────────────────────────────────────────────────────────────────────────────────

Saved in TWO places (for reliability):

    Place 1: api_level_complete() in app.py
        When level 3 is finished and the AJAX call succeeds,
        Flask immediately writes to players.json.

    Place 2: win() route in app.py
        When the browser visits /win, Flask checks if the player
        was already saved. If not, saves them again.
        This is the backup — guarantees the name is always saved
        even if the AJAX call failed.

    Duplicate prevention:
        session["saved"] = True is set after first save.
        Second save only happens if session["saved"] is False.

    Data saved per player:
        {
            "name":      "Captain Name",
            "birthday":  "2000-01-01",
            "completed": true,
            "date":      "2026-03-28",
            "stats": [
                {"level":1, "time":45, "mistakes":2, "efficiency":0.8, ...},
                {"level":2, "time":80, "mistakes":4, "efficiency":0.6, ...},
                {"level":3, "time":120,"mistakes":6, "efficiency":0.5, ...}
            ]
        }

10. HOW TO RUN THE GAME
────────────────────────────────────────────────────────────────────────────────

    Step 1: Open terminal in this folder

    Step 2: Install dependencies (first time only)
        pip install flask numpy
        py -3.13 -m pip install torch matplotlib
